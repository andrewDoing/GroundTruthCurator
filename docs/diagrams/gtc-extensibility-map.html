1 · Computed Tag Plugins
CT
ComputedTagPlugin backend/app/plugins/computed_tags/ · Auto-discovered
Active auto-discover
The most mature plugin system. Computed tags are automatically generated metadata derived from item content. Plugins are auto-discovered by scanning the computed_tags/ package — any Python module containing a ComputedTagPlugin subclass is registered at startup.
class ComputedTagPlugin: tag_key: str # e.g. "answer:no_answer" or "dataset:_dynamic" def compute(doc) → str | None # return tag value or None to skip

2 · Plugin Packs
PP
PluginPack backend/app/plugins/packs/ · Manually registered
Partial manual register
Named bundles of domain behavior that layer on the generic host. Packs are the broadest extension point, offering 7 hookable surfaces. Currently only RagCompatPack is registered, and only approval hooks + stats contribution are wired at runtime.
class PluginPack: name: str def validate_registration() # startup contract check def collect_approval_errors() # approval-time validation def collect_approval_waivers() # waive generic rules def get_stats_contribution() # pack-owned metrics def get_explorer_fields() # extra explorer columns def get_import_transforms() # shape import data def get_export_transforms() # shape export data
Hook Surface Status
Active validate_registration()
Active collect_approval_waivers()
Active get_stats_contribution()
Unused get_explorer_fields()
Unused get_import_transforms()
Unused get_export_transforms()
Current Implementation
RagCompatPack
Owns RAG/retrieval domain: reference management, per-tool-call retrieval state, approval waivers for retrieval-only items, and refs→per-call migration.
Not auto-discovered. To add a pack, edit pack_registry.py and register the new PluginPack instance manually.
3 · Trace Adapter Plugins
TA
TraceAdapterPlugin backend/app/plugins/adapters/ · Auto-discovered
Dormant auto-discover
Import/conversion plugins for foreign trace payloads. Adapters convert raw external data into AgenticGroundTruthEntry objects. Auto-discovered like computed tags, but registers classes (not instances) since adapters need runtime constructor args.
class TraceAdapterPlugin: name: str def adapt_payload(payload, **kwargs) → list[AgenticGroundTruthEntry]
TraceExportAdapter
Maps trace-export payloads into GT entries: history, tool calls, context, feedback, trace IDs, metadata. Tags: source:trace-export, workflow:agentic-rca.
Registry exists but isn't consumed. Current code instantiates TraceExportAdapter directly in demo_seed.py rather than going through the adapter registry. The auto-discovery infrastructure is ready for use but dormant.
4 · Export Registry
EX
ExportProcessor & ExportFormatter backend/app/exports/registry.py · Manually registered
Active manual register
Two complementary registries for the export pipeline. Processors transform normalized doc dicts before formatting. Formatters serialize docs into bytes/string payloads. Wired explicitly in container.py — no auto-discovery.
class ExportProcessor(ABC): name: str def process(docs) → list[dict]
MergeTagsProcessor
Merges manual + computed tags into a unified tags field
class ExportFormatter(ABC): name: str def format(docs) → bytes | str
JsonItemsFormatter
Raw JSON array output
JsonSnapshotPayloadFormatter
Envelope with schemaVersion, snapshotAt, datasetNames
Processor chain is configurable via EXPORT_PROCESSOR_ORDER in config. resolve_chain(requested, default_order) determines the pipeline. Formatters support factory registration for parameterized construction at runtime.
5 · Tool-Call UI Extensions
TC
ToolCallExtensionRegistration frontend/src/registry/ · Module side-effect
Active module side-effect
Registry of React components rendered inline within tool-call detail views. Extensions register by discriminator (e.g. toolCall:retrieval) and optional matches() predicate. Multiple extensions can match a single tool call. Each renders inside a PluginErrorBoundary for fault isolation.
interface ToolCallExtensionRegistration { discriminator: string; // e.g. "toolCall" or "toolCall:search" component: React.ComponentType; displayName: string; matches?: (tc) => boolean; // optional predicate filter }
RAG References Extension
Adds inline reference-management UI to retrieval-like tool calls. Matches: search, retrieval, lookup, fetch, query, find, get_documents, vector_search, and compound names.

6 · Explorer Extensions
EE
ExplorerExtension frontend/src/registry/ExplorerExtensions.ts · Module side-effect
Partial module side-effect
Plugin packs can contribute extra table columns and filter dimensions to the Questions Explorer view. Extensions register by packName. Re-registering the same pack replaces its bundle.
interface ExplorerExtension { packName: string; columns: ExplorerColumnExtension[]; filters: ExplorerFilterExtension[]; }
Built-in: rag-compat
Column: referenceCount
Renders a "Refs" column showing getItemReferences(item).length
Filter: hasReferences
yes / no filter for items with references
Surface Status
Active Column extensions → rendered in QuestionsExplorer
Unused Filter extensions → defined but no UI consumer found
DI & Bootstrap Wiring
Each extension mechanism has a different relationship to the dependency injection container and startup lifecycle:
Computed Tags
Module singleton. Lazy init via get_default_registry(). Called ad-hoc from services and routes. Not stored on the DI container.
Plugin Packs
Full DI wiring. Stored as plugin_pack_registry on the container. Validated during startup_cosmos().
Trace Adapters
Module singleton (unused). Auto-discovery exists via get_default_adapter_registry() but is not wired into the container or called at runtime.
Export Registries
Explicit DI. Built in container.py via factory methods. Passed to SnapshotService.
Tool-Call Extensions
Module side-effect. Self-registers on import of registry/index.ts. No explicit boot code.
Explorer Extensions
Module side-effect. Self-registers when ExplorerExtensions.ts exports are imported.
How to Add Extensions
+
New Computed Tag
  1. Create a new .py file in plugins/computed_tags/
  2. Subclass ComputedTagPlugin
  3. Define tag_key property and compute(doc) method
  4. Done — auto-discovered at next startup. Zero-arg constructor required.
+
New Plugin Pack
  1. Create a new .py in plugins/packs/
  2. Subclass PluginPack, implement hooks
  3. Edit pack_registry.py to import and register
  4. Wire any frontend counterpart in ExplorerExtensions.ts
+
New Trace Adapter
  1. Create a new .py in plugins/adapters/
  2. Subclass TraceAdapterPlugin
  3. Implement name and adapt_payload()
  4. Auto-discovered — but wire a consumer to use the registry.
+
New Export Format
  1. Create processor in exports/processors/ or formatter in exports/formatters/
  2. Subclass ExportProcessor or ExportFormatter
  3. Register in container.py build methods
  4. Update EXPORT_PROCESSOR_ORDER config if needed
+
New Tool-Call Extension
  1. Create a React component accepting ToolCallActionProps
  2. Create a registration file importing toolCallExtensions
  3. Call toolCallExtensions.register({...})
  4. Import the file from registry/index.ts for side-effect init
+
New Explorer Extension
  1. Define column/filter specs in ExplorerExtensions.ts
  2. Call registerExplorerExtension({...})
  3. Column extensions render automatically in QuestionsExplorer
  4. Filter extensions need a UI consumer (not yet wired)
